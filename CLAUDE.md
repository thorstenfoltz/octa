# CLAUDE.md

Guidance for Claude Code when working in this repository.

## Commands

```bash
cargo build              # debug build
cargo build --release    # release build
cargo run -- file.parquet
cargo test
cargo test <test_name>
cargo clippy
cargo fmt
```

## Installation

`sudo ./install.sh` for system-wide on Linux, `./install.sh ~/.local` for user-local. `install.bat` on Windows. Linux build needs `libgtk-3-dev libxcb-render0-dev libxcb-shape0-dev libxcb-xfixes0-dev libxkbcommon-dev libssl-dev libfontconfig1-dev libfreetype6-dev`.

## CI Pipeline

PRs to `master` run three jobs: `test` (fmt + clippy + cargo test, one shared `Swatinem/rust-cache@v2` target dir), `licenses` (`cargo deny check licenses`), and `megalinter` (shell, security, markdown lint via stock `ghcr.io/oxsecurity/megalinter-rust:v9`). Clippy and rustfmt live in the `test` job, not in megalinter, because the megalinter container has no Rust cache and ran ~1000s/PR there.

`release.yml` (tag-push) intentionally does **not** re-run `cargo test`: PR CI already validated the merged commit. Adding it back would double release wall-time for nothing.

## CLI

`octa` is **flag-driven**, not subcommand-driven: pick one of `--schema`, `--head`, `--tail`, `--sample`, `--convert`, `--sql`, `--export-schema` (`-e`), `--compare-schemas`, `--diff`, `--describe`, `--validate-schema`, `--unique-columns`, `--mcp` (mutually exclusive via clap `group = "action"`); with none, the binary launches the GUI with the positional file list. Implemented in `src/cli/` as one file per action for drop-in extensibility. Global `-f / --format {tsv|json|csv}` (default tsv) routes through `src/cli/output.rs`; `--convert` and `--export-schema` ignore it (extension-driven / raw-code output). `--export-schema` selects its target via `-t / --target` (clap `SchemaTargetArg` → `octa::data::schema_export::SchemaTarget`). `disable_help_flag = true` + a custom `--help` with `ArgAction::HelpLong` makes `-h` and `--help` print the same long-form output.

`--validate-schema` is the one action whose exit code carries a non-error signal: `1` on a successful read where the schemas drift (so CI pipelines can gate on it). `cli::dispatch` pulls it out of the `Result<(), _> → ExitCode` mapping so `validate_schema::run` returns `Result<ExitCode, _>` directly.

`--mcp` is the one action that does **not** route through `cli::dispatch`: `main.rs::run_mcp` peels it off before dispatch so the GUI and other-CLI paths never construct a tokio runtime.

Man page source: `docs/cli/octa.1.adoc` (AsciiDoc). The release workflow installs asciidoctor, renders `octa.1`, and bundles it; `install.sh` falls back to rendering on the fly. The mkdocs site mirrors the page at `docs/cli/man-page.md`; drift between the two is the cost of dual-discoverability.

Adding a new action: append a flag to `cli::Cli`, add an `Action` variant, drop `src/cli/<verb>.rs`, extend `Cli::detect_action`, and add a match arm in `cli::dispatch`.

Global `--rows N|all` overrides the streaming initial-load cap for one invocation (parsed via `cli::parse_rows_flag`; `all` → `usize::MAX`). `cli::dispatch` installs an `octa::formats::InitialLoadRowsGuard` at the top so every action handler sees the lifted cap.

`--sql` grew workspace flags so a single invocation can JOIN across formats and persist its result. `--sql-table NAME=PATH` (repeatable) registers extras as additional workspace tables. `--sql-attach ALIAS=PATH` (repeatable) ATTACHes a DuckDB / SQLite file under `ALIAS` so every inner table is reachable as `alias.schema.tbl`. `--sql-write-to PATH` with `--sql-write-table TABLE` (plus optional `--sql-write-schema SCHEMA` for DuckDB and `--sql-write-mode {create|append|replace}`) persists the SELECT result instead of printing it; the success line `wrote N row(s) to <path> · <schema>.<table>` goes to stdout and the table payload is suppressed. `src/cli/sql.rs` builds a fresh `SqlWorkspace` for every invocation and tears it down at exit.

## MCP server

`octa --mcp` runs a stdio JSON-RPC server (rmcp 1 + tokio current-thread). Code lives in `src/mcp/` with one file per tool under `tools/`. Tools: `read_table`, `tail`, `sample`, `schema`, `list_tables`, `count_rows`, `run_sql`, `convert`, `export_schema`, `profile`, `find_duplicates`, `value_frequency`, `search`, `compare_schemas`, `diff_tables`, `describe_file`, `validate_against_schema`, `unique_columns`, `write_table`, `edit_table`. `profile` wraps `run_query(table, "SUMMARIZE data")`; `find_duplicates` / `value_frequency` / `search` / `compare_schemas` / `diff_tables` / `describe_file` / `validate_against_schema` / `unique_columns` are thin wrappers over the pure library functions in `src/data/` (no library changes). `tail` slices the last N rows of the loaded window; `sample` returns a seeded random sample via `octa::data::sample::sample_table`; `diff_tables` wraps `octa::data::diff::diff_rows` and returns the rows unique to each side as `table_to_json` payloads plus a `shared_keys` count. When adding a tool whose response returns rows, reuse `tools::table_to_json`. `validate_against_schema` is a thin shell over `compare_schemas` (after `parse_json_schema` re-hydrates the expected columns); `Timestamp(...)` columns normalise to `Timestamp(Microsecond, None)` on the round trip because JSON Schema can't carry the unit/timezone tuple.

- Tool descriptions are inlined as string literals at the `#[tool(description = ...)]` site, because rmcp's macro doesn't accept a `const &str` there.
- `schemars` is pinned to `1` to match what rmcp re-exports; a 0.8 dep lands the derive on the wrong trait and `Parameters<T>` fails at compile time.
- Caps: `mcp_default_row_limit: Option<usize>` (default `Some(1000)`, `None` = unlimited), `mcp_default_cell_bytes: usize` (default 65,536, `0` = no cap). Per-call `limit`: `Some(0)` = unlimited, `None` = default. Per-call `unlimited: bool` installs an `InitialLoadRowsGuard` inside `spawn_blocking` to lift the file-loader cap; `limit` only slices the response (combine both to return every row). Responses surface `truncated`, `total_rows_available`, `cell_truncated`; oversized cells get a `[truncated: N bytes; ...]` marker.
- Settings are read **once at server startup**; a running `octa --mcp` does not pick up GUI changes. Defaults are duplicated literally in `src/ui/settings.rs::{default_mcp_row_limit, default_mcp_cell_bytes}` because settings live in the library and `src/mcp/` lives in the binary.
- Blocking work runs on `tokio::task::spawn_blocking`. `main.rs::run_mcp` installs `tracing_subscriber` against stderr (JSON-RPC owns stdout) and prints a single-line ready banner reporting the resolved caps.

Adding a new tool: drop `src/mcp/tools/foo.rs` with a `Params` struct + `pub async fn handle(...)`, register it in `tools/mod.rs`, and add a wrapper method on `OctaMcpServer` with `#[tool(description = "...")]`.

`run_sql` grew the same workspace surface as `--sql`: `extra_tables: Vec<{name, path, table?}>` registers additional files as workspace tables (name is sanitised via `sanitize_sql_name`), `attach: Vec<{alias, path}>` ATTACHes whole DuckDB / SQLite files for the call (extension picks the kind), and `write_to: Option<{path, schema?, table, mode, create_schema_if_missing?}>` persists the SELECT result instead of returning rows. When `write_to` is set the response shape becomes `{ kind: "write_back", rows_written, created_schema, target }` and the row payload is omitted. Each call builds a fresh `SqlWorkspace` inside `spawn_blocking` and drops it at the end — workspace state never leaks across MCP calls.

`write_table` / `edit_table` are the data-write tools (distinct from `convert`, which only transcodes, and `run_sql write_to`, which only targets DuckDB/SQLite). Both reuse the format registry's `write_file` and two new inverse-of-`table_to_json` helpers in `tools/mod.rs`: `cell_from_json(value, arrow_type)` and `build_data_table(columns, rows)`. `write_table` takes inline `columns` (name + optional Arrow `type`, default `Utf8`) + array-of-arrays `rows` and writes any *file* format by output extension; `mode` is `create` (errors if the path exists), `overwrite`, or `append` (file must exist, column names must match, rows added to the end). DB files are rejected as `write_table` targets (their `write_file` needs `db_meta`). `edit_table` edits an *existing* file in place: `set` (cells, `col` is index or name), `insert_rows` (`at` defaults to append), `delete_rows` (applied highest-index-first); it reads through the native reader, applies the mutations, then `apply_edits()` + `write_file()` — the GUI save sequence — so SQLite/DuckDB keep their diff-based save (only changed rows UPDATE/INSERT/DELETE). Column changes are not supported.

## Performance settings

- `initial_load_rows` (default **5M**) caps the first batch for streaming readers. Exposed as a process-wide `AtomicUsize` (`src/formats/mod.rs::INITIAL_LOAD_ROWS`); `set_initial_load_rows` runs from `OctaApp::new`, the Settings apply path, *and* `run_mcp` so changes take effect without restart for the GUI and are applied at MCP startup. `InitialLoadRowsGuard` is an RAII override used by CLI `--rows N|all` and per-call MCP `unlimited: true` — swap the atomic, restore on Drop. Safe only because the CLI is single-threaded and the MCP server uses a current-thread tokio runtime that serialises tool dispatch. `AppSettings.initial_load_rows_unlimited: bool` (Settings → Performance "Unlimited" checkbox) overrides the numeric with `usize::MAX`.
- `syntax_highlight_max_bytes` (default 1024 KB) gates the raw editor's syntect pass. Stored as raw bytes; the Settings dialog exposes a Bytes / KB / MB unit picker so users don't read raw counts.
- `text_mode_extensions: Vec<String>` forces unknown extensions through `TextReader`. Checked in `OctaApp::load_file` *before* the registry lookup; user input is lowercased and stripped of a leading dot. The file picker's "All Supported" filter is unioned with this list.
- `thousands_separators_in_cells` (default **on**) toggles grouping of numeric cells in the table view; `number_separator_style: SeparatorStyle` (English / European) picks the grouping + decimal-mark chars (European decimal comma applies even with grouping off). Display-only: handled entirely in `src/ui/table_view/rows.rs` via `octa::data::num_format::format_cell_number`; never touches saved/exported/CLI/MCP output.
- `excel_max_auto_sheets` (default **5**) caps how many Excel sheets open automatically (each in its own tab); above it, `load_file` raises the multi-select `pending_sheet_picker` instead.
- `trim_whitespace_on_load` (default **on**) + `warn_on_whitespace_trim` (default **on**) drive the load-time whitespace pass (see Architecture).
- `offer_repair_on_malformed` (default **off**) opts into the malformed-CSV/TSV repair prompt (see Architecture).
- `language` (default `"en"`) drives `octa::i18n` (see i18n).

## Licensing

MIT. The `licenses` CI job enforces every transitive license against `deny.toml`'s allowlist; copyleft (AGPL/GPL/LGPL/SSPL) is excluded. Two attribution artifacts:

- `THIRD_PARTY_LICENSES.md`: per-crate index, regenerated via `cargo about generate about.hbs --output-file THIRD_PARTY_LICENSES.md`.
- `licenses/<SPDX-id>.txt`: canonical text per SPDX identifier, hand-curated. When a new license family enters the dep tree, add a file.

`install.sh` / `install.bat` ship `THIRD_PARTY_LICENSES.md`, `LICENSE`, and `licenses/` alongside the binary.

## Architecture

Native desktop GUI for tabular data, built with `eframe` / `egui` (immediate-mode), Rust edition 2024. Requires a C compiler for native deps (duckdb, rusqlite, etc.). The crate is both a library (`src/lib.rs` re-exports `data`, `formats`, `ui`) and a binary (`src/main.rs`); tests `use octa::data::*`.

The canonical format registration order lives in `FormatRegistry::new()` (`src/formats/mod.rs`); the user-facing list of supported formats lives in `docs/getting-started/supported-formats.md`. Both update together when adding a format. Read-only formats (SAS, R, HDF5, NetCDF, EPUB, GeoJSON, Fixed-Width) omit `supports_write` and fall through to the trait default of `false`.

`TextReader` claims every source/config extension Octa can highlight (`.py`/`.rs`/`.go`/`.ts`/web markup/...) so they're advertised as supported, not just opened via fallback. The `highlight_whitelist_is_supported` test pins `HIGHLIGHT_WHITELIST` and the registry's extension set together. `src/formats/sniff.rs::sniff_format` adds **content-based detection** (magic bytes + JSON/CSV probes) for files with a missing or wrong extension: `reader_for_path` consults it before the Text fallback, and `load_file` retries via the sniffed reader (`try_content_sniff_reload`) when the extension-chosen reader errors. `reader_by_name` maps a sniff result back to a reader.

### Module Layout

- `src/main.rs`: app entry + root state (`OctaApp`); top-level UI orchestration; routes interactions back to the data model.
- `src/data/mod.rs`: `DataTable`, `CellValue`, `ColumnInfo`. Edits live in a separate `HashMap<(row, col), CellValue>` overlay; the original `rows` only mutate when `apply_edits()` is called. `structural_changes` flags non-cell edits. `BinaryDisplayMode` controls how `CellValue::Binary` is rendered. `db_meta: Option<DbRowMeta>` carries database row identity (`table_name`, `row_tags: Vec<Option<i64>>`, `original`, `original_columns`) parallel to `rows`; new rows have `None` tags.
- `src/data/search.rs`: `RowMatcher` for Plain / Wildcard / Regex search-filter-replace.
- `src/formats/mod.rs`: `FormatReader` trait + `FormatRegistry`. Multi-table sources implement `list_tables` and `read_table(name)`; single-table formats inherit defaults that fall through to `read_file`.
- `src/formats/parquet_reader.rs`: `build_arrow_array` and `data_type_from_string` map between our string type names and Arrow `DataType`. Both must update together when adding a new Arrow type.
- `src/formats/sqlite_reader.rs`: `rusqlite` (bundled). Stores each `rowid` in `db_meta.row_tags`. Diff-based writes: DELETE missing tags, INSERT `None` tags, UPDATE only when cells differ from `original`. Schema changes rejected.
- `src/formats/duckdb_reader.rs`: `duckdb` (bundled). No native rowid, so a synthetic `__octa_row_id BIGINT` column is added on first read and stripped from the user-visible schema. Diff-on-save matches SQLite.
- `src/formats/spss_reader.rs`: SPSS (read-only) via `ambers`, which requires arrow ^57; the project pins `arrow57 = { package = "arrow", version = "57" }` alongside arrow 58 until ambers updates.
- `src/formats/{sas,stata,rds,dbf,hdf5,netcdf,geojson,epub}_reader.rs`: pure-Rust, mostly read-only. RDS is the tabular subset only (root `data.frame`/`tibble`). `geojson_reader` claims **only `.geojson`** (JsonReader keeps `.json`).
- `src/formats/fwf_reader.rs`: fixed-width (`.fwf`/`.prn`), read-only, best-effort. `infer_field_ranges` finds column boundaries by scanning a sample of lines for char positions blank in every line (the gaps); first line is the header (blank field names become `col_N`). All columns Utf8. Pure `read_fwf(&str, &Path)` for testability. `epub_reader` uses `rbook` + `htmd` (Apache-2.0; the `epub`/`html2md` crates are GPL-3.0, blocked by `deny.toml`).
- `src/sql/mod.rs`: re-exports the workspace surface (`SqlWorkspace`, `AttachKind`, `WriteTarget`, `WriteMode`, `WriteReport`, `QueryKind`, `QueryOutcome`, `sanitize_sql_name`). `run_query(table, query)` stays as a one-shot wrapper that builds a fresh workspace, registers the table as `data`, and tears it down — every legacy call site keeps working.
- `src/sql/workspace.rs`: `SqlWorkspace` owns one persistent DuckDB connection plus `tables` (name -> origin) and `attachments` (alias -> file). API: `new`, `set_active_table` (the `data` TEMP TABLE), `add_table_from_{file,datatable}`, `attach`, `detach`, `remove_table`, `list_attached_tables`, `execute`, `write_result_to_db`. DuckDB ATTACH native for `.duckdb`/`.ddb`; SQLite ATTACH tries `INSTALL sqlite; LOAD sqlite` then falls back to per-table `SqliteReader` (`alias__table`). SQLite *writes* always go through `rusqlite` (extension never required). `execute` result columns are all `Utf8` (matches `run_query`). Sync; MCP wraps with `spawn_blocking`.
- `src/ui/table_view.rs`: virtual table renderer. Manages scroll, resize (min 60 px, no upper cap), drag-reorder, multi-cell / row / column selection, inline editing, context menu, optional row line breaks, binary display mode. Returns an `Interaction` struct; the caller does all mutations.
- `src/ui/toolbar.rs`: top toolbar (logo, File / Edit / View / Search / Help menus, zoom, view-mode radio, custom title-bar buttons). Wrapped in `egui::MenuBar` so adjacent menu buttons auto-switch on hover. Returns a `ToolbarAction` struct.
- `src/ui/directory_tree.rs`: sidebar directory browser, docked left / right per setting. Each row is a `Sense::click()`-allocated full-width rect so the whole row is the click target; `read_sorted_dir` is the sole disk walker.
- `src/ui/table_picker.rs`: modal for picking a table on multi-table sources (single-table DBs auto-load). Explicit window id `octa_table_picker_dialog_v2` (keeps persisted size off the title-derived default); height fit-to-content up to `table_picker_visible_rows` (default 10). Body/footer split via `Panel::bottom` + `CentralPanel`, not a hand-computed `body_h` (a px-off estimate made the `Resize` widget grow the window every frame).
- `src/view_modes/`: per-mode renderers (Notebook, Markdown, Raw, JSON / YAML Tree, SQL, Compare, EPUB, Map). JSON and YAML trees share `render_value_tree` parameterized by a private `TreeKind`; `markdown::render_pulldown` is `pub(crate)` so the EPUB view reuses it. The Notebook view's **source cells are editable** (`render_notebook_view` takes `&mut TabState` + `readonly`): each cell is a frameless `TextEdit::multiline`, code cells keep syntect highlighting via a `.layouter`, and changes write back through the normal `tab.table.set` overlay (so undo / save / modified marker all work). The `.ipynb` writer (`jupyter_reader::write_notebook`) re-reads the original via `source_path` and overwrites only `source`/`cell_type`, preserving outputs / `execution_count` / metadata.
- `src/view_modes/map.rs`: GeoJSON Map view via `walkers`. Walkers spawns its own tokio runtime in a private thread, so the eframe app stays sync. Polygon holes are stroked but not cut out of the fill (egui has no even-odd rule).
- `src/view_modes/text_ops.rs`: `apply_case_to_selection` (Upper / Lower) shared by SQL and raw editors. Uses char-to-byte range mapping to mutate the TextEdit buffer in place.
- `src/ui/syntax.rs`: syntect highlighter wrapping `SyntaxSet::load_defaults_newlines()` + bundled `assets/Terraform.sublime-syntax`. `HIGHLIGHT_WHITELIST` deliberately excludes formats with dedicated views (JSON / YAML / XML / Markdown / TOML / CSV / TSV). Size-gated by `syntax_highlight_max_bytes`.
- `src/ui/settings.rs`: `AppSettings` + TOML persistence (`~/.config/octa/settings.toml`, `~/Library/Application Support/Octa/settings.toml`, `%APPDATA%\Octa\settings.toml`). `IconVariant::Random` resolves to a concrete colour at startup; the picked colour is stored on `OctaApp.resolved_icon` once in `OctaApp::new` so the window icon, toolbar logo, and welcome screen stay in sync across the session. The **Rainbow easter-egg theme** is the one exception: while `ThemeMode::Rainbow` is active, `ensure_logo_textures` ignores `resolved_icon` and renders `assets/octa-random.svg` instead. Leaving Rainbow (Settings dropdown or the toolbar theme-toggle) nulls `logo_texture` / `welcome_logo_texture` so the user's configured icon comes back on the next rebuild.
- `src/ui/status_bar.rs`: bottom status bar. The navigation input field jumps to `R5:C3`, `R5`, `C3`, a row number, or a column name on Enter. Accepts a `busy: bool` + `busy_hint` for a 12 px Spinner during background loads / updates.
- `src/ui/theme.rs`: `ThemeMode` and `ThemeColors`. `apply_theme(ctx, mode, font_size)` is the single entry point.

### Key Design Patterns

- **Interaction structs**: UI components return plain structs of options / bools; `main.rs` reads them and mutates state. No callbacks.
- **Edit overlay**: cell edits live in `DataTable.edits` until `apply_edits()`. Structural mutations (`insert_row`, etc.) shift edit indices atomically.
- **Lazy filter**: `filter_dirty` triggers `recompute_filter()` before rendering. `filtered_rows: Vec<usize>` maps display index to data index.
- **Selection model**: cell click selects one cell; **Ctrl+click a cell** toggles it in a disjoint `selected_cells` set (the first toggle promotes the previously selected cell into the set). Row-number click selects a row (Ctrl/Shift extends); column-header click selects a column. Copy honors whatever is selected.
- **Tab multi-selection for Compare**: Ctrl-click on a non-active tab toggles `OctaApp.tab_multi_selection`. The `CompareSelectedTabs` shortcut (default F9) runs when exactly one tab is selected. `begin_compare_with_tab` clones the right side's state from memory rather than re-loading from disk so in-memory edits carry through.
- **Right-click copy**: TextEdit-backed views (raw, SQL) show **Copy** (greyed when no selection, reads selection by id) + **Copy All**. Non-TextEdit views (Markdown, JSON tree, notebook labels) keep one whole-content entry since `Label::selectable(true)` already handles partial selection + Ctrl+C.
- **Arrow navigation**: extend-selection shortcuts (Ctrl+Arrow) are resolved before plain arrows in `draw_table`, so Ctrl+Arrow never also moves the cursor. `JumpFirstRow / JumpLastRow / JumpFirstCol / JumpLastCol` are Ctrl+Shift+Arrow. **`ScrollPageUp` / `ScrollPageDown`** (default Ctrl+PageUp / Ctrl+PageDown) step the selection by `floor(data_area_height / row_height)` rows and let `scroll_row_into_view` follow — spreadsheet-style page navigation. All bindings remappable via `ShortcutAction`.
- **Zoom**: Ctrl+Plus / Ctrl+Minus / Ctrl+0 in 5% steps, 25-500%. `OctaApp.zoom_percent` is transient.
- **Clipboard**: dual-path, `arboard` for OS + `TableViewState.clipboard` fallback. TSV cells, newline rows.
- **Lazy row loading**: Parquet loads the first 5M rows (`initial_load_rows`) sync, then `bg_row_buffer`/`bg_loading_done`/`bg_can_load_more` background-stream the rest (UI drains per frame). The native reader (`parquet-58`) rejects > 32,767 row groups; `parquet_reader.rs::read_file` retries via a DuckDB path (`read_parquet` + `query_arrow`, reusing `arrow_value_to_cell`). Both honour `initial_load_rows` and strip **pandas index columns** (`__index_level_0__` + names in the Arrow `pandas` metadata's `index_columns`) via `pandas_index_columns`.
- **Column data types as strings**: `ColumnInfo.data_type` holds Arrow type names (`"Utf8"`, `"Int64"`, …). All readers must produce strings that round-trip through `data_type_from_string()`.
- **Unsaved changes guards**: window close and file open both check `is_modified() || raw_content_modified` and show Save / Don't Save / Cancel.
- **Read-only mode**: `OctaApp.readonly_mode` is a transient session flag (NOT persisted), toggled by F8 or **View → Read-only mode**; every editing path funnels through `OctaApp::is_readonly()`. First toggle queues `ReadOnlyNotice` unless opted out via `show_readonly_notice`. Status bar shows a plain `[Read-only]` text pill (the lock glyph is tofu in egui's font).
- **JSON / YAML tree key editing**: `render_value_tree` overrides `selection.bg_fill` to a high-contrast neutral; the theme default is accent-tinted and collapses against accent-colored keys. Object keys are renamable inline (Enter commits via `json_util::rename_object_key_at_path`, which rebuilds the map to preserve insertion order); array indices are not. Edits flow back to `tab.raw_content` via the kind's serializer.
- **Parse-error fallback**: `OctaApp::load_file` calls `fallback_to_raw_text` when a text-format reader returns `Err`. The file reloads via `TextReader`, the active tab switches to `ViewMode::Raw`, and `tab.parse_error_banner` shows a dismissible orange banner. Files > 500 MB skip the fallback. Binary formats (Parquet, SQLite, …) never fall back because raw bytes render as garbage.
- **Column-wide date promotion**: `run_date_inference_pass` promotes string columns whose values are *all* consistent with one date/datetime layout (ISO, dotted European, slashed European/US, dashed US, ...) to `Date`/`DateTime`. Ambiguous columns (e.g. `02/03/2024`) queue onto `pending_date_pickers` for a per-column European/US/leave-as-text choice. Binary readers excluded (already typed). Candidate pruning falls out of `NaiveDate::parse_from_str` rejecting month > 12.
- **Multi-file open**: `File → Open` uses `pick_files()` and CLI accepts multiple paths. All paths land on `pending_open_queue`; `drain_pending_open_queue` pops one per frame and pauses while a `pending_table_picker` or `pending_date_pickers` modal is up.
- **Raw CSV quote / escape modes**: `format_delimited_text` in `view_modes/raw_text.rs` runs a quote-aware tokenizer (delimiter inside `"a,b"` stays one field; RFC 4180 defaults `Double` + `Doubled`). `colored_layouter` reuses its `split_delimited_line_ranges` so coloring respects quotes. Coloring + alignment flipped off via the slow-file prompt for CSV/TSV > 10 MB.
- **Best-fit column width**: double-click on a column-header seam fires `compute_optimal_col_width` (measures header + cells via `layout_no_wrap`, capped at 5,000 sample rows, sets `col_widths[col]` = max + 16 px). Rust 2024 edition required for `cargo fmt` (bare `rustfmt` without `--edition 2024` reorders imports wrong).
- **Parse in new tab**: scope (Cell / Row / Column / Whole table) + format (text-style only: JSON / JSONL / YAML / TOML / XML / CSV / TSV / Markdown / Plain Text) → a `tempfile::NamedTempFile` named with the format extension, routed through `OctaApp::load_file`. The new tab's `source_path` is cleared so Save doesn't silently overwrite `/tmp`.
- **Value Frequency dialog**: `Ctrl+Shift+I`, header right-click, or **Analyse → Value frequency…**. With no column context, a column picker (`value_frequency_pick` -> `src/app/dialogs/value_frequency_picker.rs`) is raised. Compute in `octa::data::value_frequency::compute_value_frequency` (testable); UI `src/app/dialogs/value_frequency.rs`. Numeric binning: `BinningMode::Sturges` (clamped `[5,30]`) or `Custom(n)` (`[1,1000]`), equal-width via `fixed_bins`. **Binned results keep every bucket (incl. empties), ascending, NOT truncated by `top_n`**; raw mode sorts by count desc, honours Top-N. Controls drawn in a `Panel::top` *before* `freq` so changes take effect same frame. Non-finite floats surface as trailing rows. "Filter table to this value" writes into `column_filters`.
- **Numeric display formatting**: `src/data/num_format.rs` is the single source - `format_cell_number(value, Option<NumberFormat>, thousands, SeparatorStyle) -> Option<String>` (`None` for non-numeric) + `round_value`. `NumberFormat { decimals: Option<i32>, rounding }` is `Eq`; **decimals is signed** (positive = digits after point, negative = round before, `None` = Auto). `SeparatorStyle::{English,European}` picks chars. Per-column formats on `TabState.column_number_formats: HashMap<usize, NumberFormat>` (session-only), threaded into `draw_table` -> `rows.rs`. Configured via header right-click or **Edit -> Number format...**, both opening `src/app/dialogs/column_format.rs` — movable/resizable (explicit window id `octa_column_format_dialog_v2`; `Panel::bottom` footer + `CentralPanel` body so resize works vertically), edits **live** (decimals via persistent buffer `column_format_decimals_buf`). The dialog carries an in-dialog **multi-column picker**: `TabState.column_format_cols: Vec<usize>` (seeded from the selected numeric columns on open) lists every column the one format applies to; the live-apply loop writes the format to all of them and removes it from any column unchecked that frame. **Rounding is display-only**: on Save if any column `rounds_values()`, `do_save_tab` defers to `pending_round_save` + `round_save_prompt.rs` (writes a `round_table_in_place` clone; live table keeps precision).
- **Whitespace trim on load**: `octa::data::trim::trim_string_columns(&mut DataTable) -> Vec<String>` strips leading/trailing whitespace from string cells **and column titles** in place (interior kept) and returns the changed column names. `OctaApp::run_trim_pass` runs it in `apply_loaded_table` *before* date inference when `trim_whitespace_on_load` is on; for DB-backed tables it re-syncs `db_meta.original` + `original_columns` so trimming isn't seen as edits / a schema change on save. When `warn_on_whitespace_trim` is on and columns changed, sets `pending_trim_warning: Option<TrimWarning>`, rendered as a dismissible banner in `central_panel.rs` next to the date-format banner.
- **GUI glyphs are ASCII-only**: egui's bundled font renders em-dash, en-dash, `->`, `...`, `|` source glyphs (`—`/`→`/`…`/`·`) as tofu squares. Keep UI strings (and prose, per user preference) ASCII; this also covers `\u{2014}`-style escapes in string literals.
- **Excel multi-sheet**: `ExcelReader` implements `list_tables` (one `TableInfo` per worksheet, columns from the header row) + `read_table(sheet)` (shared `read_sheet` helper) and overrides `FormatReader::opens_all_tables() -> true`. `load_file` branches on `opens_all_tables`: ≤ `excel_max_auto_sheets` opens every sheet in its own tab (reads all while the reader borrow is live, then applies); above it raises `pending_sheet_picker` (`src/app/dialogs/sheet_picker.rs`, multi-select, first N pre-checked, no upper limit). DB readers keep the default single-select `pending_table_picker`. Excel *write* is still single-sheet (`write_excel`).
- **Find duplicates**: **Search → Find duplicates…** or **Ctrl+Shift+D** (`ShortcutAction::FindDuplicates`). Compute `octa::data::duplicates::find_duplicate_rows` (text keying via `CellValue::to_string()` joined with `\x1F`, sorted). UI `src/app/dialogs/find_duplicates.rs`. Two modes (`FindDuplicatesMode`): `Highlight` marks rows `MarkColor::Orange` (clear via **Edit → Mark → Clear all marks** / `DataTable::clear_all_marks`); `NewTab` clones into a fresh `TabState`. Keys seeded from selection. Plain `HashMap<String, Vec<usize>>`; `int(1)` vs `float(1.0)` don't dedupe (Display strings differ).
- **Archive viewer**: `.zip`/`.tar`/`.tgz` -> `octa::formats::archive_reader::ArchiveReader` (read-only); `read_file` returns one row per entry (`path`, `size_bytes`, `compressed_bytes`, `mtime`, `is_dir`, `type`); `format_name` starts with `"Archive"` for detection. Action bar `src/app/archives.rs::render_archive_action_bar` shows when `active_tab_is_archive()`; **Open selected entry** -> `extract_entry_bytes` + tempfile + `OctaApp::load_file_in_new_tab` (pushes an empty placeholder `TabState` so `apply_loaded_table`'s `current_empty` heuristic never clobbers the listing tab). Failures route to the archive tab's `parse_error_banner`. Tempfile leaked via `.keep()`. `.tar.gz` not routed (ambiguous vs `.csv.gz`; rename to `.tgz`); `.tar.bz2` skipped (no allowlisted bz2 crate).
- **Multi-search**: **Search → Multi-search…** or **F6** (`ShortcutAction::MultiSearch`). Pure `octa::data::multi_search::search_table(...) -> Vec<MultiSearchHit>` (reuses `RowMatcher`). Binary side `src/app/multi_search.rs` owns `MultiSearchState` (results + worker + cancel in `Arc` atomics), bottom `egui::Panel`. Scopes: `AllOpenTabs` (sync) and `Directory` (worker on `read_sorted_dir`, single level, hidden skipped, per-file cap `grep_max_file_size_mb` default 50 MB). Parse failures non-fatal: worker pushes `SkippedFile { reason: Oversized|ParseError }` into a `skipped` arc, surfaced as a separate "N skipped" chip (distinct from `last_error` fatals). Result rows `source · row · col — snippet`; click jumps (existing tab via `find_tab_by_path` else `load_file` + `focus_cell_in_active_tab`). Caps 10k total / 1k per file.
- **Schema Export**: **File → Export schema…** or **F7** (`ShortcutAction::ExportSchema`; F7 not Ctrl+Shift+X which collides with `Event::Cut`). Dialog opens on `SchemaTarget::ALL[0]` (Postgres); a chip row switches targets. Library under `src/data/schema_export/{mod,sql,pydantic,typescript,json_schema,rust}.rs` — pure `(&[ColumnInfo], &str) -> String`. Nine targets: five SQL dialects (Postgres/MySQL/SQLite/Databricks/Snowflake) behind one `Dialect` enum in `sql.rs`; Pydantic/TypeScript/JSON Schema/Rust one file each. Three surfaces: GUI (`src/app/dialogs/schema_export.rs`), CLI `--export-schema`/`-e` (`src/cli/export_schema.rs`, `-t/--target`), MCP `export_schema`. Idents via `sanitize_ident`+`is_safe_ident` (bare when safe else quoted per dialect; Pydantic/Rust emit alias/rename). Unknown Arrow types -> TEXT + comment via `unknown_marker`. Table name from file stem (`data` unsaved), PascalCased for class targets, `Row_` prefix for numeric. Adding a target: `pub fn export(...)` (new file or `Dialect` arm), append a `SchemaTarget` variant + four match arms + the `SchemaTargetArg` (CLI) / `Target` (MCP) mirrors.
- **Chart tab** (`ViewMode::Chart` + `TabState.is_chart_tab`): pure `build_chart(...) -> Result<ChartPrep, ChartError>` in `src/data/chart.rs`. Kinds: `Histogram` (Sturges), `Bar` (categorical, aggregated, `ChartLimits.max_categories` default 200), `Line` (X-sorted), `Scatter`, `Box` (Tukey 5-number, whiskers clamped to 1.5*IQR). Axis bounds (`x/y_min/max/step`) on `ChartConfig`, wired through `Plot::default_*_bounds` + `*_grid_spacer`. `sample_indices` honours `ChartLimits.max_points` (Bar/Box fold full input). `cell_to_f64` coerces Date->days, DateTime->epoch secs. It's a *tab, not a view mode*: `OctaApp::open_chart_tab` clones into a fresh `TabState` (`is_chart_tab`, `source_path = None`); `available_view_modes` returns `[Chart]`. Trigger: Analyse -> Chart or `OpenChart` (F5). UI in `src/view_modes/chart.rs` (control bar + Customise collapsible + Export PDF/PNG/SVG). Bar X axis uses `Plot::x_axis_formatter` for category names. `series_styles: HashMap<usize, SeriesStyle>` keys on source column index so styles survive reordering. Export in `src/data/chart_export.rs`: SVG -> `resvg` PNG / `svg2pdf` PDF (shared SVG). Deps: **egui_plot 0.35**, **svg2pdf 0.13**.
- **Parse in new tab — header preservation**: Cell/Row/Column scopes build a **synthetic `DataTable`** (source column name(s) as headers, data as string cells) then serialize via the registry — same path as Whole-Table, so headers survive every format. Plain Text is the exception (no schema, verbatim cells). Helpers in `src/app/dialogs/parse_in_new_tab.rs`: `synthetic_table`, `build_synthetic_payload`, `serialize_table_bytes`.
- **Formula evaluation diagnostics**: `evaluate_formula_with_diagnostics` returns `FormulaOutcome { value, bad_cell }`. Non-numeric strings / Date / DateTime / Binary / Bool / Nested cells become `Err(FormulaBadCell)` rather than coercing to `0.0`; the first bad cell short-circuits the row. `add_column.rs` shows a `Formula skipped N of M row(s)…` banner via `tab.parse_error_banner`.
- **Custom title bar (opt-in)**: when `use_custom_title_bar` is true, `main.rs` strips system decorations and `draw_toolbar` renders close / max / min buttons at the right edge of the *existing* toolbar (no separate panel). Drag-to-move falls back to the WM convention (Alt+drag / Super+drag on most Linux desktops). Takes effect after restart since `with_decorations` is a viewport-builder flag.
- **Cycle view mode (F4)**: advances `tab.view_mode` through `available_view_modes`. Gated on the TextEdit-focus check so F4 in the SQL / raw editor types the key normally.
- **Malformed-file repair** (opt-in `offer_repair_on_malformed`, default off): on a CSV/TSV load `maybe_offer_repair` runs `csv_reader::analyze_delimited` (bad UTF-8, BOM/control chars, delimiter mismatch, ragged rows) and, if flagged, raises `pending_file_repair` -> `src/app/dialogs/repair_file.rs` (Repair / Open-raw / Cancel + preview). `resolve_file_repair` reloads via `read_delimited_opts` (`ReadOptions { lossy_utf8, delimiter, strip_bom_controls }`). CSV/TSV only.
- **Date/Time calculation**: pure `octa::data::time_calc` — `evaluate_cell(op, &CellValue[, &CellValue])` for `TimeCalcOp::{Difference, AddSubtract, ConvertDuration, Extract}` (parses via `date_infer`, arithmetic via chrono; month/year add clamps day to month length). **Edit -> Date/Time calculation...** opens `src/app/dialogs/time_calc.rs` (`TimeCalcDialog` state), materialising a new column like `add_column.rs`.

### Write Support

Most formats support writing. CSV tracks its delimiter (`csv_delimiter`) so saves preserve the original separator. Excel writes via `rust_xlsxwriter` (calamine is read-only); only `.xlsx` round-trips. ODS has a dedicated reader+writer that hand-rolls the OpenDocument 1.2 package (`mimetype` stored uncompressed + `META-INF/manifest.xml` + `content.xml`). TOML / YAML serialize through a JSON intermediary.

### Database write semantics (SQLite / DuckDB)

DB writes are **diff-based, never overwrite**. The reader snapshots each row's identity (`rowid` / synthetic `__octa_row_id`) and original cell values into `db_meta`. On save, in one transaction: DELETE original tags missing from `row_tags`, INSERT rows with `None` tag, UPDATE rows whose cells differ. Schema changes (column rename/add/drop) are rejected before touching the file — do those in another tool.

### SQL view

Active table is exposed as DuckDB temp table `data`. Every tab owns its own long-lived `SqlWorkspace` (built lazily, stored as `TabState.sql_workspace: Option<SqlWorkspaceHandle>` — a newtype around `Rc<RefCell<SqlWorkspace>>`), dropped when the tab closes. The Workspace section lists registered tables + ATTACH-ed DBs with per-row [×]/[detach], [+ Add table…], [Attach database…]; the [refresh] next to `data` re-pushes the active table's edits (SQL sees a cached snapshot until refresh). Result actions `[Run]`/`[Clear]`/`[Export…]`/`[Write result to DB…]`; the last opens `SqlWriteBackDialog` (`src/app/dialogs/sql_write_back.rs`) -> `SqlWorkspace::write_result_to_db`. Writing into the open file's own `db_meta.table_name` with Append/Replace is rejected (points at regular Save). Panel docks Bottom/Top/Left/Right via Settings -> SQL; a nested `TopBottomPanel` resizes editor vs result. Editor has a monospace line-number gutter sharing one `ScrollArea` with the TextEdit. `tab.sql_query` empty; template `SELECT * FROM data LIMIT {sql_default_row_limit}` via `TextEdit::hint_text`. Autocomplete chips (SQL keywords + column names + workspace tables + `alias.schema.tbl`) appear at word-token end. Upper/Lower consumes its combo via `consume_key` (avoids egui's `Ctrl+U`); `SelectAllRows` (Ctrl+A) scopes to a focused TextEdit via `load_state`. Workspace state is **session-only**.

### Undo / Redo

Dispatched from `src/app/shortcuts_dispatch.rs` via `OctaApp::do_undo` / `do_redo` (`src/app/edit_ops.rs`), which also reset `filter_dirty` and `widths_initialized`. Gated on no TextEdit being focused so Ctrl+Z in the SQL / raw editor undoes *text*, not table state. `UndoAction` covers cell edits, structural mutations, and colour marks; structural ops push to `undo_stack` and clear `redo_stack`.

### Colour Marking

`DataTable.marks: HashMap<MarkKey, MarkColor>` with `MarkKey::{Cell, Row, Column}`. Precedence: cell > row > column. Toolbar **Edit → Mark** and the `Mark` shortcut (Ctrl+M default) apply the configured `default_mark_color` to the current selection with the precedence `selected_rows > selected_cols > selected_cells > selected_cell`. The right-click context menu uses the same precedence as long as the right-clicked target is part of the current selection (multi-cell / row / column); outside the selection it falls back to colouring the single clicked target. `interaction.set_mark` / `clear_mark` carry `Vec<MarkKey>` so call sites can batch the multi-key apply in one go. Rainbow easter-egg theme: marked-cell text colour is pinned to `Color32::WHITE` (or `BLACK` for the pale-yellow mark via `MarkColor::needs_dark_text`) because `colors.text_primary` cycles every frame and would otherwise crash into the mark fill.

### Auto-Update

Toolbar update checker. `ureq` for HTTP, `flate2` / `tar` / `zip` for archive extraction. Logic in `src/main.rs` and `src/ui/toolbar.rs`.

### Testing

Integration tests in `tests/`. Binary fixtures (parquet, avro, arrow, xlsx) are auto-generated on first run by `tests/common/mod.rs::ensure_fixtures()`. Text fixtures (csv, json, …) are checked in. DB tests seed each test's database in a `tempfile::NamedTempFile` directly via `rusqlite` / `duckdb`. SQL tests build `DataTable` literals and call `octa::sql::run_query` directly. `tests/sql_workspace_tests.rs` exercises the `SqlWorkspace` surface — multi-table JOINs, ATTACH against DuckDB and SQLite (with and without the bundled sqlite extension), name collisions, and write-back round-trips for both targets in every mode (`Create` / `Append` / `Replace`).

### Windows Build

`build.rs` uses `winresource` to embed manifest, icon, and metadata. `windows/octa.exe.manifest` controls UAC + DPI.

### Containers

Headless `Dockerfile` (multi-stage: `rust:1-bookworm` builder -> `gcr.io/distroless/cc-debian12` runtime) for CLI + `--mcp`. The GUI libs (GTK/X11) compile but are never dlopen'd headless, so the runtime drops them; the binary only needs glibc + libstdc++ + liblzma. `docker run -v $PWD:/data octa --schema /data/f.parquet`; MCP over stdio with `-i ... --mcp`. Podman-compatible (same Dockerfile). Docs `docs/cli/docker.md`. Alpine/musl rejected (DuckDB/rusqlite bundle C++).

### i18n

Hand-rolled `src/i18n.rs` (not a derive crate, to span the lib/bin split): `t(key)` + `set_language(code)`, TOML catalogs embedded from `locales/*.toml` (English master + 12 Latin-script langs; Roboto covers the accents). `set_language` runs at `OctaApp::new` and on Settings apply (live, no restart), driven by `AppSettings.language`. The `every_language_covers_every_english_key` test enforces key parity. String migration to `t()` is **incremental** — only the toolbar menus are migrated so far; add a key to *every* locale then call `t()`. Cyrillic/CJK/Arabic deferred (need bundled fonts + RTL).
